哪吒监控面板从安装到接入Cloudflare

发表于2023-07-27|更新于2023-09-01|未知|Linux
|字数总计:1.1k|阅读时长:3分钟|阅读量:

前言

  1. 哪吒监控是一个开源、轻量的服务器和网站监控、运维工具

  2. 对CPU、内存、存储、网络带宽、网络流量等一系列服务器指标的监控

  3. 对第三方IP、Web服务进行ICMP、TCP、SSL证书状态进行持续监控

  4. 支持WebSSH、定时任务、批量命令下发等运维功能…

  5. GitHub地址：https://github.com/naiba/nezha

服务端安装

要安装哪吒监控，你必须拥有：

  1. 一台能够访问GitHub与Docker的服务器

  2. 一个接入Cloudflare的域名与SSL证书

  3. 一个Github账号，用于哪吒监控面板的用户鉴权

  1. 注册一个域名，这里使用的 Freenom 家的免费域名（有条件最好还是使用收费的域名，免费域名的DNS服务器经常出现波动，会导致探针失联）

  2. 将域名托管到 Cloudflare （免费提供CDN加速、网络优化、安全防护、网络分析以及应用等服务），这一步看个人意愿，可以不操作

  3. 添加A记录，解析域名，并验证。Ping 域名返回的 IP 与实际 IP 不同，说明 CDN 代理成功。

  4. 创建Oauth2 应用

  5. 登录GitHub后，访问Developer settings

  6. 点击New OAuth App

  7. 如下图所示，填写以下信息

  8. Homepage URL: 你的域名，在本例中为https://home.443.lol/

  9. Authorization callback URL: 你的域名+/oauth/callback，在本例中为https://home.443.lol/oauth/callback

  10. 点击创建 Client secrets ，然后记录下 Client ID 和 Client secrets，备用

安装哪吒面板服务端

  • 输入1并按回车键，开始安装

  • 根据提示输入Client ID、Client secrets、GitHub ID，标题和端口号自定义即可。

配置Nginx服务器

  1. 在本例中，我们将假设您已经拥有已经安装完成的Nginx服务器并已配置好SSL证书。

  2. 如果没有，请自行安装宝塔面板或Nginx Proxy Manager。

  3. 下述配置任选其一完成，不可同时安装。

宝塔面板反向代理配置

  1. 在宝塔面板中为域名添加反向代理，任意填写代理名称与目标并保存。

  2. 然后点击配置文件按钮，删除所有内容并粘贴下述配置。

  3. 注意将端口修改为您在安装哪吒面板时配置的相应端口。

  4. 在本例中，20001对应面板访问端口，20002对应RPC端口。

Nginx Proxy Manager配置

  1. 添加反向代理，填写相应主机与端口信息。

  2. 注意SSL标签页中需要开启HTTP2支持

  3. Advanced页面中，填入以下配置：

    注意将端口与主机名修改为您在安装哪吒面板时配置的相应端口

  4. 至此 哪吒监控面板安装完成。尝试访问您的域名，应该可以看见面板正常工作。

配置哪吒面板

  1. 在浏览器中打开您的哪吒面板，并使用GitHub账号登录。
  2. 登入后进入设置页面，找到 未接入CDN的面板服务器域名/IP 设置项
  3. 填入您的域名，按下保存按钮 。

安装监控端

  1. 登录哪吒面板网页-管理后台-主机

  2. 点击添加服务器按钮，填写相应信息后点击添加

  3. 添加完成后，在页面中复制一键安装命令并在您想要监控的服务器终端中执行

  4. 在被监控的服务器内执行与上面相同的命令，选择 8 安装监控Agent。

  5. 填入服务端主机 IP 解析的域名（不可以套CND），然后填入对应的服务秘钥。

完成之后

恭喜，您已成功完成所有配置。
